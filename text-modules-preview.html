Section Front Module Library

Eight layout modules for composing section fronts. Modules 5–8 are text-first — visual hierarchy from PT Serif headlines, the red kicker, whitespace, and rule lines, not imagery. Resize the browser or open on your phone (iOS Safari / Android Chrome) to preview mobile layouts. All modules are mobile-first.

Implementation overview — Newspack native vs. custom CSS vs. custom build

# Module Effort Implementation path
1Hero / LeadCSShomepage-articles (1 post, mediaPosition top) + custom sizing CSS. Falls back to Module 6 if no image.
2Featured + compact listCustomColumns block wrapping two homepage-articles instances + CSS for 60/40 split. Stacks on mobile automatically.
3Equal-card gridNativehomepage-articles, postLayout:"grid", columns:2–3. Works out of the box — already used by the 3 live section fronts.
4Horizontal carouselCustomhomepage-articles + CSS scroll-snap override via block wrapper class. Basic swipe needs no JS. Desktop reverts to static row.
5Pure headline listCSShomepage-articles, showImage:false, postLayout:"list" + CSS for kicker-column layout. 2-col newspaper variant adds columns: CSS only.
6Text-forward cardCSShomepage-articles, showImage:false, postLayout:"grid" + CSS for 3px red left bar via ::before pseudo on block wrapper.
7Small-thumbnail rowCSShomepage-articles, mediaPosition:"left" + CSS to constrain image to 64–72px square. No-image variant = red bar at same width.
8Featured + text listCustomGroup block: one homepage-articles (1 post, large) + one homepage-articles (showImage:false, list). CSS for 2px rule divider.

Module 4 — Horizontal Carousel · touch-swipe on mobile/tablet · static 4-col row on desktop

📱 Mobile: horizontally scrollable with scroll-snap (one card per snap point). Cards are 260px wide — partial next card is visible at the right edge, signalling that more items exist. Uses -webkit-overflow-scrolling: touch for iOS momentum scroll. The scrollbar is 3px and barely visible (decorative only). No JS required for swipe behavior. Desktop: static 4-column grid — overflow: visible, snap disabled.

Module 5 — Pure Headline List · kicker + headline + byline · no image ever · likely the most-used module

📱 Mobile: kicker stacks above headline (no two-column layout below 640px). Lead headline clamps 21px → 28px. Each item is 44px+ tall. No horizontal overflow at 320px — PT Serif wraps gracefully at any width. The 2-column newspaper variant only kicks in at 1024px+ and collapses back to single column below — no breakage on tablet or phone.
📰 Design intent: visual hierarchy from the kicker column (90px fixed, like a broadsheet section tag), the PT Serif headline weight, and the thin rule rhythm — not from any image. Red kicker + serif headline + muted byline = three typographic voices that establish editorial tone with zero imagery. NYRB / broadsheet newspaper feel.

Module 6 — Text-Forward Card · red left bar · headline leads · no image expected or needed

📱 Mobile: single column. Padding 18px → 22px desktop. Red left bar stays 3px at all sizes. Headline clamps 19px → 26px. Excerpt is 3-line clamped, never overflows. Min-height 44px guarantees comfortable tap. The wide featured variant (6b) is always full-width — no change needed on mobile, it's already single column.
📰 Design intent: the 3px red bar is the design — not a missing-image indicator. The bar earns its place on opinion pieces, long-form reviews, interviews. Headline is 5–7px larger than an image card's because it carries the visual weight alone. White card surface against the off-white page creates contrast with no image needed. Especially strong on mobile where the full-width text is immersive.

Module 7 — Small-Thumbnail Row · 64–72px image accent · text dominant · works with or without image

📱 Mobile: thumbnail 64px (72px on tablet+). Gap 13px → 16px. Headline is 3-line clamped — no overflow at 320px. No-image fallback (3px red bar) stays at the same horizontal position as a thumbnail, so the text block starts at the same x-offset with or without an image — no layout shift. Full row height is one tap target.
📰 Design intent: good for sections with mixed coverage (30–50% of posts have images). Text takes 85% of the row at any viewport size. Works like a magazine digest list. On mobile, the small thumb or the red bar provides a visual left anchor that makes the list easy to scan quickly.

Module 8 — Featured + Text List · one lead (text or image) above compact headline list

📱 Mobile: lead card full-width, list stacks below. The 2px ink rule (the strongest visual break in the module) is visible and correct at all sizes. List items use kicker-above-headline layout below 640px, kicker-column layout at 640px+. Lead headline clamps 22px → 34px. List items 44px+ tall. The image variant (8b) stacks correctly: image fills width at 16:7 ratio, then text below.
📰 Design intent: editorial weight earns the lead slot — not having a photo. The 2px rule between lead and list is the sharpest break on the module. "One big, several small" is a broadsheet front-page structure collapsed into a single reusable module. The same module works whether the lead has an image or not — no fallback needed, no layout shift.

Module 9 — Symmetric 3-Column Lead Block · text list | image anchor | text list · both side cols text-only, no images · image-poor archive friendly

📰 Structure: Both side columns are identical text-only headline lists (kicker + headline + byline, ~4–5 items, thin horizontal rules). Center is the single visual anchor — image + bigger headline + dek. Thin vertical rules divide the 3 columns. Symmetric and readable with zero images in the side columns — intentionally distinct from Pitchfork's asymmetric layout.
📱 Responsive: mobile = single column, center story first, then left list, then right list. 3-col layout at 1024px+.

Section Geometric Marks — Design Directions

Three directions, each showing all four section marks as a coordinated family. Do not apply to live site until a direction is chosen.

Direction A — Paired Bars

Two bold rectangles arranged differently per section. The arrangement logic echoes WEEKLY's strong horizontal letterforms — each mark feels like a graphic logotype fragment. Graphic and editorial, no curves anywhere.

A La Music
Photography
Food & Drink
Out N About

Direction B — Double Vision

One geometric form per section — rendered twice: filled solid in front, same shape as an open outline offset 5px behind. Creates layered depth. Each section gets a distinct base form (square / portrait rect / landscape rect / diamond).

A La Music
Photography
Food & Drink
Out N About

Direction C — Chamfered Shard

Same 24×24 solid rectangle for every section — but each has a different corner clipped at 45°. The clip location is the differentiator: top-right / bottom-left / top-left / both-top-corners. Badge-like and slightly raw, feels crafted not computed.
