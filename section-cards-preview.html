Preview A — Three card variants in one mixed grid

🤖 Automatic — no editorial action needed: At render time, PHP calls wp_get_attachment_image_src( get_post_thumbnail_id(), 'full' ) and reads the returned width. Width ≥ 1024px → card--large (full-bleed 16:9). Width < 480px → card--small (framed, padded). No _thumbnail_id → card--text. The two thresholds live in one PHP constant — adjustable without touching templates.

Curation approach — what's automatic vs editorial vs code

⚠️ Important context before reviewing the table: This table assumes the custom PHP template approach (category.php). Whether to build that at all, or to use Newspack's own Homepage Articles block + Query Loop instead, is a foundation decision that should be settled before any template code is written. See the architectural comparison in your response for the full breakdown.
Decision Who handles it How
Image quality tier (large / small / none) 🤖 Automatic wp_get_attachment_image_src() at render — no manual step across 2,800 posts
Card type (image / text-forward) 🤖 Automatic Derives from whether _thumbnail_id exists — already set by Option C
Which posts appear on a section front ✏️ Editorial Standard WP category assignment — add/remove a category on any post from the post editor
Section front layout (image-grid vs list) ✏️ Editorial Category meta dropdown — WP Admin → Categories → Edit → Layout. No code.
Lead story (hero slot) ✏️ Editorial WordPress sticky post — one click from the post editor sidebar. Fallback: most recent post with a large image auto-leads if nothing is sticky.
Posts per page ✏️ Editorial Category meta — same screen as layout. Default 12 for grid, 15 for list.
Adding a new section layout type (e.g. masonry, compact) ⚠️ Code edit New template-part file + new dropdown option. One-time dev task per new layout — not routine. Rare.
Adding a new main section to the theme ⚠️ Code edit Add category slug to $main_sections in category.php. 30-second edit — but it is a code edit. Could be avoided by storing the list in WP Options.
Changing image quality thresholds ⚠️ Code edit Two PHP constants in functions.php. Could be exposed as a WP Options field if this matters long-term.
Per-section presentation that differs significantly (e.g. Music wants a sidebar filter) ⚠️ Code edit Beyond layout-type switching, genuinely different UI patterns need a developer. This is the boundary of what categorymeta can handle.
